# PLG2627 — フル更新ビルド仕様（2026/27・全20クラブ）

各クラブ担当エージェントは **このファイルを最初に読み、次に `arsenal.html`（監督続投テンプレ）と `chelsea.html`（監督交代＝移籍マーキング多めテンプレ）を Read して、その構造・CSS・クラスを"完全に踏襲"**して自分の担当ページを書く。デザインは変えない。変えるのは「チーム固有の色・データ・選手配置・文章」だけ。

## 絶対ルール
1. **1ファイル完結HTML**。`arsenal.html` と同じ `<head>`（Google Fonts CDNリンクそのまま）、同じ `<style>` 構造、同じセクション順（HEADER → INFO STRIP → TACTICAL BRIEFING〔lead＋守備/保持の2 mini-pitch＋role-grid〕→ LEGEND → NOTES〔risk-card / incoming-card〕→ FOOTER）。クラス名・レイアウトは一切変えない。
2. **チームカラー**: `:root` の `--arsenal` 系3変数（`--<team>`, `--<team>-light`, `--<team>-glow`）を担当クラブの色に置換し、`body` の radial-gradient や badge のグラデも同様に差し替える。ただしCSS変数名は既存のまま（例: `--arsenal`）にして値だけ変える＝他クラスが壊れない。`--gold` 系はそのまま。
3. **選手名は全てカタカナ**。同名は必ずイニシャル/通称で区別（例: ブルーノ・G / B・フェルナンデス）。**主将に (C)**。
4. **EAFC26 の OVR レーティング**（数字）を各 `.mp-disc` に入れる。SoFIFA / FUTBIN / EA公式を web で確認。確証が無ければ妥当な推定値を入れ、NOTES かフッター注記で「推定」と明記。
5. **守備時＋保持時の2フォーメーション図**を必ず両方作る（`.mini-pitch` 内に `.mp` を11個ずつ絶対配置）。保持時は可変（インバーテッドSB等）を反映して配置を変える。
6. **移籍70%基準**: 離脱70%以上=該当 `.mp` を warning ボーダー等で示し NOTES risk-card に％付きで記載／補強70%以上=incoming で示し incoming-card に記載。70%未満は NOTES に注記のみ。`.pct warn / inc / neutral` を使う。
7. **HEADER**: `DOSSIER NN / 20 · POS xx`（NNは下の担当番号、POSは25/26最終順位。昇格組は `POS —（昇格）`）。`SEASON 2026 — 2027`。formation-badge は基本布陣。manager-info に監督名（カタカナ）＋就任時期＋「続投/新任」。champions-tag は該当時のみ（優勝/CL出場/昇格 等の短いタグに読み替え可）。
8. **INFO STRIP 4セル**: 戦術アイデンティティ / 保持時の形 / 非保持時 / 25/26成績（順位・勝点・GD。昇格組はChampionship成績＋「PL昇格」）。
9. **文章は日本語**（Noto Serif JP）。lead・phase-desc・role・NOTES は担当クラブの実情に即して具体的に書く（テンプレの言い回しを流用しつつ中身は必ずクラブ固有に）。
10. FOOTER は `<TEAM> · DOSSIER NN / 20` / `EAFC 26 RATINGS` / `COMPILED FOR TAKEMOTO` を踏襲。
11. 出力先は指定の絶対パス。**書き終えたら3行以内で「完了・主要な不確実点・レーティングの確証度」だけ返す**。
